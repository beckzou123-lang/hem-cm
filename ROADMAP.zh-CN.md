# HEM-CM 路线图

这份路线图聚焦于如何把 HEM-CM 从一个已经具备完成度的本地事件智能研究终端，继续推进为一个可持续维护、可扩展的开源事件驱动研究平台。

## 当前阶段

已经落地的基础包括：

- 默认英文、支持中文切换的双语仓库与产品界面
- 全局态势台、事件作战室、资本市场作战台、高频信号中心、回测实验室与模型中心
- 基于 Prisma + SQLite 的本地优先技术栈
- 演示数据、初始化流程与验证链路
- GitHub 展示素材、Release Notes 与公开发布闭环

## 下一阶段

### 1. 信源智能

- 在 Google News、RSS、JSON feeds、静态网页之外，继续扩展更多实时信源接入
- 提升面对不一致文章布局时的抽取稳定性
- 增加信源可信度评分与更好的去重逻辑
- 支持可复用的信源接入预设，服务重复性研究工作流

### 2. 模型透明度

- 继续扩展因果母模型与子模型文档
- 在界面中提供更清晰的“为什么是这个阶段 / 为什么是这条路径”证据面板
- 为路径推演增加更明确的失效信号跟踪
- 让模型参数变化在不同版本之间更容易比较

### 3. 回放与评估

- 增加命中率、拟合度之外更丰富的回放评估指标
- 跟踪模型版本之间的预测漂移
- 优化预测结果与真实结果之间的时间线对比体验
- 加入可复用的基准事件集合，支持类似回归测试的评估方式

### 4. 产品工作流

- 支持协作批注与研究备注
- 增加保存视图与围绕 watchlist 的研究工作流
- 优化事件包与市场映射结果的导出封装
- 为首次进入项目的用户提供更明确的 onboarding

### 5. AI-Ready 工具能力

- 增加可选的证据摘要模块
- 提供更结构化的 prompt / export 界面，便于下游模型使用
- 增加适用于外部 copilots 的 agent-ready event bundle 格式
- 始终保持“可解释性优先”，而不是事后补救

## 更长期

### 1. 更广覆盖

- 增加更多事件家族与专用子模型模板
- 扩展更多资产类别与区域化市场映射
- 增加宏观与政策传导模板

### 2. 协作层

- 以讨论为中心的研究复盘流程
- 共享模型库与可复用情景模板
- 面向贡献者的新事件家族与 adapter 扩展工具包

### 3. 部署形态

- 提供可选的托管 demo 或 sandbox 部署
- 支持更偏生产环境的数据库与任务运行器
- 增加团队级认证与工作区隔离能力

## 当前最值得贡献的方向

现在最有价值的贡献包括：

- 新的信源接入
- 模型透明度增强
- 回放指标设计
- 文档与 onboarding 打磨
- 示例数据集与基准事件样本

## 路线原则

- 先可解释，再追求复杂度
- 先本地优先，再做重基础设施扩张
- 先事件推演能力，再做纯看板美化
- 先沉淀可复用原语，再做一次性工作流
