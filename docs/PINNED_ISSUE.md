# Pinned Issue Kit

Use this as the pinned community-entry issue for the repository.

## Recommended title

Wanted: source adapters, flagship event cases, and mapping feedback

## Copy-paste body

HEM-CM is now public, and this is the best place to help shape what it becomes next.

If you have spent a few minutes with the repo, the screenshots, or the local demo, I would love feedback in any of these directions:

### 1. Source adapters we should connect first

Which public feeds would make HEM-CM materially more useful?

Examples:

- Reuters / AP / official ministry releases
- SEC / exchange announcements
- central-bank or regulator feeds
- shipping, commodity, or macro signal feeds
- RSS, JSON, or webpage sources that are stable enough to parse

If you suggest a source, please include:

- the link
- why it matters
- whether it is structured (`RSS`, `JSON`, `API`) or semi-structured (`HTML`, `webpage`)

### 2. Real event loops we should model next

Which event families deserve deeper product treatment?

Examples:

- geopolitics / war
- tariffs / trade restrictions
- regulation / antitrust / platform rules
- labor strikes / social unrest
- supply-chain breaks
- major corporate accidents or governance shocks

If you suggest an event loop, tell me:

- the event or theme
- why it is high-value for capital-market research
- which market transmission paths matter most

### 3. Market mapping that feels weak, missing, or wrong

If any part of the current output feels unconvincing, I want to know.

Examples:

- sector winners / losers ranking
- priced-in level
- expectation gap
- crowding signal
- rates / FX / commodity transmission
- China vs US market translation

The most useful feedback is specific:

- which case you looked at
- which output felt off
- what you expected instead

### 4. Forecast and replay logic that needs work

If a scenario path, condition, invalidation signal, or backtest explanation feels weak, please call it out.

Useful feedback:

- which horizon looked wrong
- whether the path set felt incomplete
- whether the causal chain missed a key driver or constraint
- whether the replay / realized comparison was not persuasive enough

### 5. Good first contributions

If you want to contribute directly, reply here and mention which lane fits you best:

- source adapter
- event case modeling
- market mapping matrix
- replay / backtest metrics
- product UI / UX
- docs and onboarding

If enough people are interested, I will convert the most requested directions into scoped starter issues.

## Short version

If you only want to leave one quick comment, answer one of these:

1. Which source should HEM-CM ingest first?
2. Which real event loop should HEM-CM model next?
3. Which current output feels least convincing?

Thanks for checking out the project. If the direction resonates with you, a star helps a lot.
