## GitHub Discussion – v1.0.1 更新帖

### 标题

HEM-CM v1.0.1 已上线：一次安全补丁与仓库维护更新

### 正文

HEM-CM v1.0.1 现已发布。

这次是一次补丁版本更新，重点不在新增页面，而在依赖安全修复与仓库维护质量完善。

本次更新包括：

- 升级 `prisma` 到 `^6.19.3`
- 升级 `@prisma/client` 到 `^6.19.3`
- 修复 `@prisma/config` 与 `effect` 这条传递依赖漏洞链
- 重新验证 `build`、`validate` 与 `npm audit`
- 同步仓库内 Release 文档与 README 入口到 `v1.0.1`

为什么这次更新值得关注：

- 默认分支已经运行在修复后的 Prisma 依赖链上
- 之前报告的高危 Dependabot 告警已经关闭
- Release 元数据与仓库入口更清晰，新访客更容易理解项目状态

Release：
https://github.com/beckzou123-lang/hem-cm/releases/tag/v1.0.1

如果你正在关注 HEM-CM，我现在最想收集的反馈是：

1. 下一步最值得优先补的信源接入是什么
2. 哪个模型或回放验证环节仍然不够清晰
3. 哪类维护或 onboarding 工作最值得优先加强
