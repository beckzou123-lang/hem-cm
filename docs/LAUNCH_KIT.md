# HEM-CM Launch Kit

## One-Line Pitch

Open-source event intelligence terminal for capital markets.

## Two-Line Pitch

HEM-CM turns breaking news into explainable causal chains, scenario paths, cross-market impact maps, and replayable research workflows.

It combines event modeling, market translation, live-source intake, and backtest surfaces in one local-first product.

## Core Angles

- Explainable, not black-box
- Event-first, not just dashboard aggregation
- Cross-market, not single-asset
- Replayable, not just predictive
- AI-ready, but already useful without remote model dependency

## Launch Post – LinkedIn / Long Form

Today I’m open-sourcing HEM-CM, an event intelligence terminal for capital markets.

The goal is simple: move beyond news aggregation and make event reasoning inspectable.

HEM-CM turns an event into:

- a causal chain
- stage-aware scenario paths
- cross-market impact mapping
- replayable backtest and validation views

What’s included in this first public release:

- a reusable parent causal model plus six event sub-models
- Global Overview, Event War Room, Market Desk, Signal Monitor, and Backtest Lab
- local-first setup with Prisma + SQLite
- English-first bilingual UX with Chinese support

This is especially useful for researchers, PMs, and developers exploring explainable AI-ready financial tooling.

Repository:
https://github.com/beckzou123-lang/hem-cm

I’d love feedback on:

- which event family should be expanded next
- which source adapters matter most
- where the reasoning flow should become more transparent
