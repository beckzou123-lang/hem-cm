# HEM-CM 项目亮点梳理

## 一句话亮点

HEM-CM 不是简单的资讯聚合器，而是一套把热点事件转成可解释研究流程的事件智能终端：从信号接入、事件分类、阶段判断、因果链构建、路径预测，到市场映射与回测，全链路闭环。

## 为什么这个项目容易打动 GitHub 访客

- 它不是只有前端页面，而是有完整的事件引擎和数据链路
- 它不是黑盒 LLM 套壳，而是可解释、可复现、可继续升级的模型框架
- 它是产品化表达，不是“算法脚本 + 表格结果”
- 它默认就能跑通，开箱即见完整演示数据和真实案例

## 模型层亮点

### 1. 通用因果母模型

项目把事件统一拆解成：

- 行为体
- 目标
- 激励
- 约束
- 触发
- 响应
- 扩散
- 市场传导

这意味着项目不是按新闻标题做平铺，而是在做结构化事件理解。

### 2. 六类子模型设计

当前已内置六类子模型：

- 地缘 / 战争
- 贸易 / 政策
- 监管 / 法律
- 企业 / 产业
- 社会 / 劳工
- 科技 / 平台

每个子模型都具备：

- 专属变量
- 权重配置
- 阶段偏置
- 路径模板
- 市场映射矩阵
- 展示重点

这使得项目既有统一框架，又能保留不同事件类型的差异化逻辑。

### 3. 路径预测机制

项目不是只给一个结论，而是：

- 固定 7 / 30 / 90 / 180 / 365 天五个时间窗
- 每个时间窗生成三条路径
- 概率可解释
- 同时给成立条件与失效信号

这种输出方式很适合研究、投研、教育展示和后续 AI 增强。

### 4. 市场映射能力

同一事件可以映射到：

- 国内社会
- A 股
- 美股
- 商品
- 利率 / 汇率 / 风险偏好

并进一步拆成：

- 受益
- 受损
- 已定价程度
- 拥挤度

这让它天然具有“从事件到交易框架”的表达力。

## AI 相关设计亮点

虽然当前核心不是外部大模型驱动，但它有很强的 AI-ready 特征：

- 规则与配置分层清晰，便于后续替换或增强
- 数据是结构化存储，适合做检索、RAG、解释生成
- 信号、事件、路径、市场影响彼此解耦，便于引入 agent 或 LLM 作为增强层

后续很适合扩展的 AI 能力包括：

- LLM 辅助证据摘要
- 多语言内容自动生成
- 路径解释自动撰写
- 研究助手 / 投研 Copilot
- 外部文档抽取增强

## 产品层亮点

- 首页是“全局态势台”，不是普通表格
- 事件详情页是“作战室”，具有因果链、路径区和影响映射
- 市场页支持从已有事件和实时新事件两条链路进入
- 回测页补足了“预测之后怎么验证”的闭环
- 模型页支持在线查看和修改配置
- 设置页支持中英文切换，适合国际访客浏览
